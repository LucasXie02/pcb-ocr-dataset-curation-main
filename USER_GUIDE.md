# PCB OCR 审核工具 — 使用指南

## 启动

```bash
python ocr_review_app.py --dataset_root Data/ocr_zip_files/0311_ocr --port 5001
```

浏览器打开 `http://localhost:5001`

## 审核流程

```
选板子 → 自动勾选一致组 → 浏览确认 → 接受选中 → 处理分歧和剩余 → 完成审核 → 下一块板
```

### 第一步：选择板子

点击顶部「板子」下拉，选择要审核的板子。
- 绿点 = 已完成（candidate/ 为空，文件已全部移到 final/）
- 黄点 = 部分完成
- 灰点 = 未开始

### 第二步：自动勾选一致组

在组件总览页，点击「自动勾选一致组」。系统自动勾选所有子板间 OCR 一致且置信度 ≥ 0.8 的组。**不会直接接受**，只是帮你勾上 checkbox。

> **关于准确率：**
> - **多子板（6 拼板、10 拼板）**：同一位置的组件在多个子板上被独立检测，一致组可信度高，快速浏览即可。
> - **单子板（无拼板）**：一致组仅表示置信度高，模型可能一致地误识别（如把 `103` 全识别成 `E01`），建议逐个确认。

### 第三步：浏览确认并接受

快速浏览已勾选的组，看缩略图上的 bbox 和文字是否正确：
- **确认没问题** → 点「全部接受选中」
- **发现有问题的** → 取消该组的 checkbox，稍后单独处理

### 第四步：处理分歧和剩余

将筛选下拉切到「待审核」，逐个处理剩余的组：

- **看起来没问题** → 勾选 checkbox → 点「全部接受选中」
- **需要修改文字或框** → 点击卡片打开对比栏 → 在对比栏内编辑或点「编辑」跳转到审核页
- **图片方向错误** → 在对比栏点「批量旋转90°」或「批量旋转180°」
- **标记为"有分歧"的组** → 点击卡片查看对比栏，常见原因：
  - 多行文本的检测**顺序不同**（如 L0/L1 对调）→ 内容正确，直接点「全部接受」
  - 某个子板识别错误 → 点「接受多数」通过正确的，拒绝错误的
  - 全部识别错误 → 逐个点「编辑」修正

### 第五步：完成审核

点击「完成审核 → final/」。系统将所有已通过的 JSON + 图片从 `candidate/` 移到 `final/`。

- 只有**每条标注行都已接受/编辑**的图片才会被移动
- 有任何未审核行的图片会被阻止，弹窗列出具体图片名
- 阻止的图片需要回到审核页逐个处理后再次点击完成审核

## 四个页签

| 页签 | 用途 |
|------|------|
| **组件总览** | 主工作区。缩略图带 bbox 标注，批量勾选接受，对比栏查看子板差异 |
| **审核** | 单张图片编辑。canvas 上画框改字，旋转图片，增删标注行 |
| **仪表盘** | 审核进度和统计数据 |
| **队列** | 所有标注行的列表，可按状态筛选定位 |

## 组件总览操作

| 操作 | 说明 |
|------|------|
| 点击卡片 | 打开对比栏，并排查看该组所有子板的检测结果 |
| 勾选 checkbox | 选中该组，配合「全部接受选中」批量通过 |
| 筛选下拉 | 全部 / 待审核 / 已审核 / 有分歧 / 有Agnostic标记 |
| 排序下拉 | 按置信度升序、降序、按类别、按位置ID |
| 位置/文字下拉 | 按子板位置或文字内容筛选特定组 |
| 方向键 ← → | 在卡片间移动焦点（蓝色边框） |
| Enter | 打开当前焦点卡片的对比栏 |
| Space | 勾选/取消当前焦点卡片 |

## 对比栏操作

点击组件总览的卡片后弹出，并排显示该组所有子板实例：

| 操作 | 说明 |
|------|------|
| 全部接受 | 通过该组所有实例的所有标注行 |
| 接受多数 | 通过与多数一致的实例，拒绝异常值 |
| 批量旋转90°/180° | 旋转该组所有图片（已审核的会被跳过） |
| 单个实例「接受」 | 通过该实例的标注行 |
| 单个实例「拒绝」 | 标记该实例为异常 |
| 单个实例「编辑」 | 跳转到审核页编辑该实例 |
| OCR 输入框 | 直接修改该实例的文字，回车保存 |
| Esc | 关闭对比栏，返回组件总览 |

## 审核页编辑

| 操作 | 快捷键 | 说明 |
|------|--------|------|
| 选择模式 | S | 点击选中已有的框，拖动调整位置和大小 |
| 画字符框 | C | 在图上拖动画出新的字符框 |
| 画文本行框 | L | 在图上拖动画出新的文本行框 |
| 接受当前行 | A | 标记当前行为已审核，自动跳到下一条 |
| 删除选中字符 | Delete | 删除当前选中的字符框 |
| 导航 | ← → | 切换到上一条/下一条标注行 |
| 缩放 | + / - / 0 | 放大 / 缩小 / 重置缩放 |
| 亮度 | [ / ] | 降低 / 提高亮度 |
| 对比度 | { / } | 降低 / 提高对比度 |
| 保存 | Enter | 保存修改（仅在有未保存修改时生效） |
| 放弃修改 | Esc | 撤销所有未保存的修改 |

### 编辑文字

在顶部文本框输入正确的 OCR 文本（仅支持 0-9 A-Z）。保存时系统自动将文本框内容同步到每个字符框的标签——无需逐个改字符。

### 新增文本行

1. 点「＋ 新增文本行」
2. 按 L 切到画行框模式，在图上拖动画出文本行范围
3. 按 C 切到画字符框模式，逐个画出每个字符的框
4. 在文本框输入 OCR 文本
5. 点保存（Enter）

### 旋转图片

点「⟳ 旋转90°」顺时针旋转。旋转会同时更新所有 bbox 坐标位置。

> **注意：** 旋转只改变图像方向和框的位置，不会改变文字内容。如果图片原来是倒的，旋转 180° 后需要手动修正 OCR 文本（如 `201` 改为 `102`）。

## 数据目录结构

```
dataset_root/
  BoardName_fused/
    candidate/          ← 待审核的 JSON + PNG
    final/              ← 已通过的 JSON + PNG（完成审核后移入）
    crop_manifest.json  ← 子板分组信息（有拼板时存在）
  review.db             ← 审核事件记录（SQLite，跨板共享）
```

## 注意事项

- 「接受」只在数据库中标记审核状态，不移动文件。必须点「完成审核 → final/」才会将文件从 `candidate/` 移到 `final/`
- 编辑 OCR 文本后保存，字符框的标签会自动与文本同步（如文本改为 `103`，三个字符框自动标为 `1`、`0`、`3`）
- 组件总览缩略图上：彩色细框 = 字符检测框，绿色半透明 = 已审核行，橙色半透明 = 待审核行
- 切换板子时组件总览的筛选会重置为「全部」
- "有分歧"常见于多行文本检测顺序不同（如两个子板的 L0/L1 对调），此时文字内容相同，可直接全部接受
